# Evidence archive

<!-- sourcebound:policy register-v2 -->
<!-- sourcebound:purpose -->
Use this archive when you need to inspect a release-scoped receipt without mistaking it for a claim about the current build. It records immutable inputs and observed results, while the current documentation and release artifacts remain the authority for supported behavior.
<!-- sourcebound:end purpose -->

**[Return to the documentation index](../docs/README.md)**.

No historical reader-trial receipt is carried on the current product surface. A receipt remains
useful only when its candidate version, commit, producer, context digests, and task evidence can be
verified together. Git history preserves earlier work without presenting it as current attestation.

The [release verification guide](../docs/VERIFY_RELEASE.md) owns current artifact verification.
The repository does not infer current usability, coverage, or documentation quality from a past
trial.
